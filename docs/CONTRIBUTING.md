# Contributing Guide

Thank you for contributing! Please follow the guidelines below to ensure a smooth collaboration.

## How to Contribute
1. Fork the repository.
2. Create a feature branch: `git checkout -b feature/your-feature`.
3. Commit changes with clear messages following Conventional Commits.
4. Open a Pull Request with a description of changes and testing steps.

## Coding Standards
Refer to `docs/coding_standards.md` for style rules and naming conventions.

## Issue Reporting
- Use GitHub issues to report bugs or request features.
- Include steps to reproduce, expected behavior, and logs if available.

## Tests
- Add unit tests for new features under a `tests/` folder.
- Run tests locally with `pytest`.

## Review Process
- PRs will be reviewed by maintainers for code quality, tests, and documentation.
- Address review comments and push updates to the same branch.
